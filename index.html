About

In this project, I found a website to be improved, identified the key usability and accessibility issues, created lo-fi and hi-fi prototypes, and developed a responsive version of the website. Scroll down ⬇️ to read about more my process and the final product!

💻

Web Page

Bagel Gourmet Café ➡️ Menu

Why?

I chose this website because the design’s barely responsive for smaller screen sizes, and defies much of a user’s conceptual model when navigating the website.

🗝️

Key Issues


Responsiveness

  • When the screen shrinks, the page simply gets cut off horizontally, so that you have to scroll to the end. On a phone, the page display simply shrinks, making the lettering too small to read.

Conceptual Model

  • Some style choices make certain texts seem like links while they aren’t, and the navigation buttons change location based on the page a user is on. (My conceptual model expects it to stay in one location)
  • Clicking on the logo does not take the user back to home page, which is now a part of most conceptual models of modern websites.

Visual Design

  • Bottom buttons blend in with the background, and so does the link randomly placed at the top left corner.
  • Layout of prices are offset, and so are very hard to read.
  • Hard to tell which options belong to what headers, since the only distinctions are underlined vs. non-underlined (some times italics?)
  • All contents are shifted to the left, with very tight columns and small text. The margins are missing in some text displays, and the color scheme offers little contrast.

Accessibility

  • There is no alt text for the images, so screen readers cannot read them.
  • Only uses h4 and h5 elements, and no h1 is found.
  • Language is never identified
  • Poor color contrast.
  • There are empty headings.
  • Missing or uninformative page title.
  • No page regions are identified.
  • Non-link texts are underlined.
  • Texts overall are very small.
  • Layout tables are present: can cause reading/navigation order issues.
